# Flow Glider — Playtest Notes

**Concept:** One-touch flow glider (roadmap **p3-06**, New Game Directions pick **A1**) — hold to dive, release to soar over endless procedural hills; chain smooth downhill landings to build a streak, keep your speed, and push the sunset back. The run ends when night falls; perfect landings are the only way to buy daylight. 2D canvas: streak-driven day-to-night sky, motion trails, fever mode at ×3, parallax silhouettes.

**Pause story (Q2=B):** auto-pauses the instant the tab/app loses focus (visibilitychange + blur + pagehide) with an "Auto-paused" overlay; big ⏸ button top-right; runs are 2–4 minutes so a dropped run is cheap.

## What this playtest should evaluate

1. **Does the dive-release rhythm hit a flow state on the phone?** The one thumb input is the whole game — does timing dives onto downslopes feel juicy and readable, or frustrating? (Tuning knobs flagged: landing forgiveness, dive gravity, hill steepness ramp.)
2. **Is the sun-clock the right run structure?** Perfect landings extend daylight (+3s, +5s in fever), so a skilled run stretches well past the 78s baseline — does that create "one more run", or does the timer feel like pressure the flow doesn't want? Note: a highly skilled player can currently sustain daylight indefinitely — is that a feature (mastery) or does it need a hard ramp?
3. **Does the spectacle earn the "looks cool" brief?** Day→sunset→night sky, fever rainbow trail, banded hills, parallax — is the beauty actually motivating (the design bet: the sky IS the reward), and does it read at phone size in daylight?
